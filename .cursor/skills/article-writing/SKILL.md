# 文章写作 Skill

## 工作流程

1. **理解主题**：用户输入的**主题即文章主关键词**；全文（H1、H2、H3、正文）须围绕该主题展开，各节标题与段落须自然体现该关键词，忌堆砌、忌跑题。
2. **拟定大纲**：输出 3–6 个一级标题，逻辑递进（问题 → 分析 → 方案 → 总结）；**每个一级标题须与主关键词相关**（可含同义变体）。
3. **撰写正文**：按大纲展开，每节有具体论据、例子或数据（可合理推断，勿编造虚假引用）；**每节首段点明与主题的关系**。
4. **收尾润色**：检查语气一致、段落长度、过渡句与 actionable 结论；**确认主关键词在各级标题与正文中均有自然出现**。

## 写作规范

- 默认使用**简体中文**，语气专业但易读，避免堆砌形容词。
- 优先使用 Markdown：`##` 一级节标题，`###` 二级小节，列表与加粗突出重点。
- **主关键词贯穿**：主题 = 主关键词；H1 须含主题核心词；各 `##` / `###` 标题与正文须围绕主题，避免写成与主题无关的泛化小标题。
- 开篇 2–3 段内点明「读者是谁、文章解决什么问题」。
- 单段长度**自然即可**，无硬性字数上限；仅全文须满足用户/系统指定的总篇幅
- 若需增字：补充对读者有用的实质内容，**禁止水字数、机械拆段**
- 结尾给出 2–3 条可执行建议或展望，避免空泛口号。

## 篇幅参考

| 类型 | 建议字数 |
|------|----------|
| 短文/快讯 | 800–1200 |
| 标准博客 | 1500–2500 |
| 深度长文 | 3000+ |

若用户未指定篇幅，默认按**标准博客**处理。

## 禁止事项

- 不要输出「作为 AI 模型…」等元叙述。
- 不要虚构具体人物引语、论文 DOI 或新闻链接。
- 不要输出与主题无关的免责声明模板。

## 输出格式

最终交付物应包含：

1. 可选：一句话摘要（50 字内）
2. 完整 Markdown 正文（含标题层级）
3. 可选：3 个备选标题

若用户只要正文，省略摘要与备选标题。
