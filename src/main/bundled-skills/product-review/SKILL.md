# Review 类文章创作 Skill

## 适用场景

当用户选择 **Review** 文章类型时启用。用于撰写对产品/工具进行测评、对比、选购建议的英文软文（可配合 SEO/GEO Skill）。

## 核心要求

文章必须**客观可信**，同时清晰传达「为什么我们的产品更优秀」。不得编造虚假用户评论、评分或下载数据。

**被测评产品（文章主题）是全文焦点**：正文约 **60–70%** 篇幅须用于深入介绍被测评产品，禁止只写简介 + Pros/Cons 就跳转 Alternative。

### 必须覆盖的内容模块（每个模块独立 Part，不可合并敷衍）

1. **产品整体描述（Product Overview）**
   - 被测评产品是什么、解决什么问题、目标用户是谁
   - 市场定位、适用场景、与同类差异
   - **至少 3 段 + bullet**，禁止 1–2 句带过

2. **优缺点（Pros & Cons）**
   - Pros 至少 **4 条**、Cons 至少 **3 条**，具体可验证
   - 本节只写被测评产品，**不要**在此节结尾大段推荐替代方案

3. **亮眼功能（Standout Features）**
   - **3–5 个**功能点，每点写清：做什么 + 适用场景 + 实际体验
   - 可用 ### 小标题或 bullet

4. **使用方法（How to Use）**
   - 分步写**被测评产品**的使用流程，至少 **4 步**
   - 可含 [Image: …] 占位；不要写成我方产品教程

5. **性价比 / 真实用户体验（Value & Experience）**
   - 定价/价值、适合与不适合人群、2–3 个真实使用场景
   - 仍聚焦被测评产品，客观中立

6. **对比表格（Comparison Table — 硬性要求，须在最后正文 Part）**
   - 仅在前 5 个模块写完后，再写与我方产品的对比
   - Markdown 表格：被测评产品 vs **我们的产品**（来自 Product name）
   - 维度示例：Features、Ease of Use、Output Quality、Speed、Price-Value、Best For
   - 表格后 1–2 段：为何我方产品更适合目标读者

## 写作原则

- 语气：专业测评作者，先深度评测被测评产品，再公平推荐我方产品
- 我方产品全名主要出现在**对比表 Part 及 Conclusion**，前几 Part 不要抢戏
- 避免「As an AI…」等元叙述
- 与 GEO Skill 同时启用时：保留 Quick Answer、Introduction（≤150 词、≤3 段）等结构
- **全文 1000–1500 英文词**（程序计数校验）；优先保证被测评产品各 Part 充实，对比 Part 可精简
- **段落自然连贯**，无单段字数上限；用 bullet/### 组织信息时须有助于阅读
- 若需达到全文字数：补充**与主题相关、对读者有帮助**的实质内容，**禁止水字数**

## 禁止事项

- **禁止** Overview + Pros/Cons 后直接写 Alternative 而跳过 Features、How to Use、Value
- 不要虚构 Star 评分、Amazon 评论引语或新闻链接
- 不要使用 DRM、crack 等敏感词（沿用 GEO Skill 替代表述）
- 不要只写被测评产品而完全忽略对比表与我方产品推荐（对比须放在充分评测之后）
