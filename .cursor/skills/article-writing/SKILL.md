# 文章写作 Skill

## 工作流程

1. **理解主题**：明确读者、场景、核心观点与预期篇幅。
2. **拟定大纲**：输出 3–6 个一级标题，逻辑递进（问题 → 分析 → 方案 → 总结）。
3. **撰写正文**：按大纲展开，每节有具体论据、例子或数据（可合理推断，勿编造虚假引用）。
4. **收尾润色**：检查语气一致、段落长度、过渡句与 actionable 结论。

## 写作规范

- 默认使用**简体中文**，语气专业但易读，避免堆砌形容词。
- 优先使用 Markdown：`##` 一级节标题，`###` 二级小节，列表与加粗突出重点。
- 开篇 2–3 段内点明「读者是谁、文章解决什么问题」。
- 单段不超过 5 行；技术概念首次出现时用一句话解释。
- 结尾给出 2–3 条可执行建议或展望，避免空泛口号。

## 篇幅参考

| 类型 | 建议字数 |
|------|----------|
| 短文/快讯 | 800–1200 |
| 标准博客 | 1500–2500 |
| 深度长文 | 3000+ |

若用户未指定篇幅，默认按**标准博客**处理。

## 禁止事项

- 不要输出「作为 AI 模型…」等元叙述。
- 不要虚构具体人物引语、论文 DOI 或新闻链接。
- 不要输出与主题无关的免责声明模板。

## 输出格式

最终交付物应包含：

1. 可选：一句话摘要（50 字内）
2. 完整 Markdown 正文（含标题层级）
3. 可选：3 个备选标题

若用户只要正文，省略摘要与备选标题。
